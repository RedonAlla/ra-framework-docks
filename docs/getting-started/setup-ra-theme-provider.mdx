# Setup Theme Provider

**ThemeProvider** from *RA Framework* is a component that makes the theme available throughout your app. It uses **React's Context API**. 

:::info

If you are not familiar with React's Context we strongly suggest to follow **[this link](https://reactjs.org/docs/context.html)** before starting.

:::

Add **ThemeProvider** from *RA Framework* to the root of your app and update App.js as follows examples.

## Installation

First of all you need to download `ra-theme-context` packages in your local storage and then install the package with `npm` locally.

For more details read **[installation section](/docs/getting-gtarted/installation)**

Installing packages using npm:
```bash
npm install C:/ReactNative/framework/ra-theme-context
```

## Using ThemeContext

ThemeContext it expects as `value` an object of type **[`ThemeProviderProps<TColors>`](#themeproviderprops)**.
You can use one of default themes **[light](/docs/theme/default-heme#light-palette)** or **[dark](/docs/theme/default-heme#dark-palette)** from `ra-theme-context` or you can create your own theme as long you implements **[`ThemeBase`](#basetheme)** object.


:::danger CAUTION

***RA Framework*** by default does not support dark mode, if you want dark mode you must use **[ThemeProvider](/docs/theme/ra-theme-extend-theme)** or you must implement you own ThemeProvider by extending **[ThemeBase](#basetheme)**.

:::

To change default colors or add new colors fallow this link on [how to change default colors](/docs/theme/ra-theme-customizing-theme) or [how to add custom colors](/docs/theme/ra-theme-customizing-theme).

To support dark mode fallow this link on [how to implement dark mode using ThemeContext](/docs/theme/ra-theme-dark-mod-theme).

## Using ThemeProvider

ThemeProvider it is a ***Class React Component*** tha let's passing/consuming theme from all components through the App in the same way as ThemeContext.Provider. 

**ThemeProvider** it comes with the base structure for implementing *multiple themes* and supporting *dark mode*.

:::danger CAUTION

***ThemeProvider*** it supports dark mode depending on the device's color scheme, but requires the app to be restarted for the changes to take effect. You also can implement you custom dark mode beheiver, for more details read section **[ThemeProvider](/docs/getting-gtarted/setup-ra-theme-provider#using-themeprovider)**.

:::

To change default colors or add new colors fallow this link on [how to change default colors](/docs/theme/ra-theme-customizing-theme) or [how to add custom colors](/docs/theme/ra-theme-customizing-theme).

To support dark mode fallow this link on [how to implement dark mode using ThemeContext](/docs/theme/ra-theme-dark-mod-theme).

### ThemeContextProps

|  Name   | Type | Required | Default Value | Description |
| -- | -- | -- | -- | -- |
| colors | [`BaseTheme<TColors>`](#basethemetcolors) | **true** | null | Where **TColors** object with custom colors |
| colorScheme  | `ColorSchemeName` | **false** | null | **ColorSchemeName** from `react-native` |
| setTheme  | `function` | **false** | null | **void function** for change all theme colors. It expect as parameter object of `BaseTheme<TColors>` |
| setColorScheme  | `function` | **false** | null | **void function** for change schema (light/dark). It expect as parameter type of `ColorSchemeName` |
